Ariadne

Connector-oriented documentation maintenance system for reducing documentation drift. The current implementation accepts local change events, applies redaction, and generates reviewable Markdown documentation update proposals.

Open API Recent Proposals

What It Does

Ariadne turns a trigger such as a Git diff or webhook payload into a structured proposal with Markdown and JSON output for human review.

Main Routes

  • POST /trigger for normalized trigger envelopes
  • POST /trigger/git for direct Git-based triggering
  • POST /patches/{id}/approve for review decisions
  • GET /proposals to inspect stored proposals

Operating Model

The current implementation is local-first and PostgreSQL-backed. External platform connectors are planned extensions of the same architecture.
